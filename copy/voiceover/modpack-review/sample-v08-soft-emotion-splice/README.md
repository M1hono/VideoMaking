# v08 收敛情绪拼接样例

状态：测评语音实验。样例文案不代表真实整合包。

## 来自 v07 的反馈

- 缺少过渡情绪。
- `angry` 太用力。
- `happy` 太用力。
- 需要把探索经验积累为规则。

## 结构

| Part | 情绪 | 文本 | 目的 |
| --- | --- | --- | --- |
| 01 | 空 | 我一开始确实有点不舒服。 | 不满，但不直接炸开 |
| 02 | `angry` | 因为它明明有这么好的底子。 | 短促轻怒，只点一下 |
| 03 | `sad` | 可每到关键节点，它都急着替玩家做完决定。 | 从愤怒过渡到失望 |
| 04 | 空 | 但玩久一点以后，我开始理解它为什么这么设计。 | 进入理解 |
| 05 | 空 | 它不是没把路做好，只是把这条路设计得太确定了。 | 释怀，不用 happy |
| 06 | 空 | 所以我最后的感觉不是生气，而是有点可惜，也算接受。 | 收束判断 |

## 经验

- 对复杂情绪，不要把 `angry` / `happy` 用在整段。
- `angry` 只适合短句点一下。
- 释怀不等于开心，优先用无 emotion 的文本表达。
- 中间需要“失望”“理解”这种桥接情绪，否则听起来像硬切。
