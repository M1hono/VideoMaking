# Minecraft 开发者周报零剪辑流程

状态：栏目分镜与自动合成约束。

## 核心原则

每期只改数据和文案，不手工剪时间线。

最终视频由以下输入生成：

- 候选项目 manifest。
- 用户精修后的正文稿。
- MiniMax 语音文件。
- 自动字幕。
- 固定视觉模板。

## 节奏结构

### 0. 开头 8-12 秒

目标：告诉观众这周看什么，不做营销式开场。

画面：

- 本期标题。
- 日期范围。
- 版本标签快速掠过。
- 3 到 5 个入选项目的卡片闪过。

### 1. 快速总览 15-25 秒

目标：先建立这一周生态概况。

画面：

- 入选项目数量。
- MC 版本分布。
- 加载器分布。
- 模组 / 整合包 / 工具比例。

### 2. 主体项目 4-8 个

每个项目固定结构：

- 项目名和类型。
- 版本号和发布日期。
- 支持 MC 版本与加载器。
- 这周更新一句话。
- 为什么整合包作者该注意。

画面：

- 左侧项目卡。
- 右侧 changelog 摘要。
- 底部版本标签。
- 如果有截图，使用截图；没有截图，使用项目图标和结构化信息。

### 3. 快速带过 3-6 个

目标：保留信息广度，但不拉长视频。

画面：

- 列表式卡片。
- 每个项目 6-10 秒。

### 4. 结尾 10-15 秒

目标：总结趋势，而不是硬 CTA。

画面：

- 本周关键词。
- 下周观察列表。
- 来源说明。

## 自动化要求

- 每段语音对应一个 scene id。
- 字幕从 TTS 或 ASR 生成后再校对。
- 单个项目没有截图时，模板自动降级为信息卡。
- 音频长度决定 scene 时长，不手工拖动。
- 项目卡片文本必须有最大行数，超出自动缩短。
