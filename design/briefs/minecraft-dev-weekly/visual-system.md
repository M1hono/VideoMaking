# Minecraft 开发者周报视觉系统 Brief

状态：设计约束草案。

## 目标

做成无需手工剪辑的整合包作者周报模板。视觉要服务信息扫描，不做花哨资讯包装。

## 画幅

默认：

- 横版：`16:9`
- 分辨率：`1920x1080`
- 目标平台：Bilibili / YouTube

可扩展：

- 竖版切片：`9:16`，只做摘要，不承载完整周报。

## 画面语言

- 信息卡片为主。
- 使用 Minecraft 生态熟悉的版本标签、加载器标签、项目类型标签。
- 不做大面积炫光和营销式背景。
- 项目截图优先，缺图时使用图标和结构化信息。

## 固定组件

- 周报标题条。
- 日期范围。
- 项目卡。
- 版本标签组。
- 加载器标签组。
- changelog 摘要框。
- “对整合包作者的意义”一句话框。
- 来源角标。

## 可读性

- 任何项目名最多两行。
- changelog 摘要最多三条。
- 单屏最多展示一个重点项目。
- 快速带过段落每屏最多三张小卡。

## 后续实现方向

适合交给 Remotion 模板：

- 根据 manifest 自动生成 scene。
- 按音频时长决定 scene 长度。
- 自动绑定字幕。
- 自动渲染完整视频。
